# Mobily Recharge UX Case Study

## Project Overview

This project is an end-to-end UX case study focused on improving the prepaid balance recharge experience in the Mobily mobile application.

The study identified two major usability issues that affected users during the recharge process:

- Poor discoverability of the recharge feature.
- Lack of support for decimal recharge amounts (halalas).

The project followed the complete UX process, including Business Analysis, UX Analysis, Requirement Analysis, User Journey Mapping, Solution Design, Wireframing, and High-Fidelity UI Design.

## Problem

The existing prepaid balance recharge experience contains two primary usability issues:

### 1. Poor Feature Discoverability

Users must navigate through multiple screens before locating the recharge feature, making a routine task unnecessarily difficult.

### 2. Limited Recharge Amount Input

The recharge amount field only accepts whole-number values, preventing users from entering the exact amount required for prepaid package purchases when prices include decimal values (halalas).

## Final Solution

The redesigned experience focuses on solving the two identified usability issues.

### Solution 1

A dedicated Recharge Balance quick action was added to the dashboard, allowing users to access the recharge feature directly from the home screen.

### Solution 2

The recharge amount field now supports decimal values (halalas), enabling users to recharge the exact amount required without unnecessary overpayment.
